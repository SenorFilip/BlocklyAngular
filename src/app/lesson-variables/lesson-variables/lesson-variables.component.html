Variables

Variables are like labels, that store data, think of them as containers with labels. They let you store data to use later in a program.



To create a variable in Python, you’ll use a variable name, an equal sign (=), and a value, as shown in the figure below.


Variables can store numbers, words, and even complete sentences, such as "Hello reader, my name is Vin!".
Here's another example:

weather = "cloudy"
Here, the variable container's name is weather (left side), and its value is "cloudy" (right side.)
You need to know a few syntax rules for naming variables so Python can understand them:
  • Variable containers are case sensitive, which means that the name var is different from Var, vAr, and VAR
  • Don’t include symbols in your variable names, except for underscores (_), or you’ll get a syntax error
  • White space inside the variable name is not allowed also
  • Don’t start a variable name with a number, as in 3player. Using numbers elsewhere in a variable name is fine, as in player3
fav_num = 3 # correct
!fav_num = 3 # wrong
@fav_num = 3 # wrong
1fav_num = 3 # wrong
=fav_num = 3 # wrong
In Python code, anything after a # symbol in a statement is a comment and is IGNORED by the Python Interpreter
You don’t need to add spaces on either side of the equal sign: your program will run fine without them.
But they do make the code easier to read, so it’s a good idea to add them.
For example player='Little Vin' is equal to player = 'Little Vin' , adding space just makes them more readable.
