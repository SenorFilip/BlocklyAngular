Objects

We now have a bunch of classes, but what about putting some things into those classes? Say we have a dog named Bobo.
We know that he belongs in the class Dogs, but what do we use, in programming terms, to describe single dog called Bobo?
We call Bobo an object of the class Dogs (you may also see the term instance of the class). To “introduce” Bobo to Python,
we use this little snippet of code:

>> bobo = Dogs()
This code tells Python to create an object in the Dogs class and assign it to the variable reginald. Like a function,
the class name is followed by parentheses.

Functions
But what does the bobo object do? Well, nothing at the moment. To make our objects useful, when we create our classes,
we also need to define functions that can be used with the objects in that class. Rather than just using the pass keyword immediately
after the class definition, we can add function definitions.

When we define a function that is associated with a class, we do so in the same way that we define any other function, except that we indent
it beneath the class definition.

>> class ThisIsMySillyClass:
def this_is_a_class_function():
print('I am a class function')
def this_is_also_a_class_function():
print('I am also a class function. See?')
We can add characteristics to each class to describe what it is and what it can do. A characteristic is a trait that all
of the members of the class (and its children) share. For example, what do all animals have in common? Well, to start with, they all breathe.
They also move and eat. What about mammals? Mammals all feed their young with milk. And they breathe, move, and eat.
We know that dogs bark, and like all mammals, they feed their young with milk, breathe, move, and eat food. When we add these characteristics to
our tree diagram, we get something like this:



Because bobo is an object, we can call (or run) functions provided by his class (the Giraffes class) and its parent classes. We call
functions on an object by using the dot operator and the name of the function. To tell Bobo the dog to move or bark, we can call the
functions like this:

>> bobo = Dogs()
>> bobo.move()
>> bobo.bark()
Variables
Functions are like actions, but Classes have something that can describe each object called variables. For example, the Dogs class
variable can be number_of_fleas, fur_color, weight, name...
>> class Dogs:
name = ""
weight = 0
fur_color = ""
number_of_fleas = 0

def __init__(self, name, weight, color, flea_number):
self.name = name
self.weight = weight
self.fur_color = color
self.number_of_fleas = flea_number

def description(self):
desc_str = "%s has %s kg. His fur color is %s and he has %s fleas." % (self.name, self.weight, self.fur_color, self.number_of_fleas)
return desc_str

>> bobo = Dogs("Bobo", 10, "brown", 0)
>> print(bobo.description())
Bobo has 10 kg. His fur color is brown and he has 0 fleas.

  • The __init__ method is something called constructors. Constructors are used to initialize the object’s state.
    The task of constructors is to initialize(assign values) to the data members of the class when an object of class is created.
